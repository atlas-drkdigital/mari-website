## PHASE 2 SYNTHESIS — the 4 cross-cutting patterns (this is the real answer to "why it feels all over the place")

**P1 — The skills run ~1 session BEHIND local reality, and the handoff ledgers UNDER-REPORT.** Every lane
found unmerged 07-17 deltas; mari-project's handoff literally says "Pending: empty" while real drift sits
unstaged. Root cause: work → local docs → handoff → skill has a lagging last hop, and sometimes the handoff
step is skipped too. **This is the maintenance failure, and it's structural, not carelessness.**

**P2 — DUAL SOURCES OF TRUTH cause the drift.** Same fact authored in two places that then diverge:
mari-project sprint vs MANAGER ACTIVE QUEUE (day-ordering, already disagree — worst one); stack table
(CLAUDE.md ↔ stack.md); "CMS is a product surface" ×3; mari-core facts vs mari-website composed copy. FIX
pattern site-wide: **one owner per fact; everyone else POINTS, never restates.**

**P3 — Some stale content will ACTIVELY MISLEAD a future build** (the dangerous class, fix first):
weak Figma-precedence rule (→ builds toward Figma), "compress before upload" (→ destroys masters),
atlas `pages.md` requiring the retired `faq` type (→ recreates a dead type), mari-website `/boat` URL +
stale boat.md nodes (→ builds against a deleted design).

**P4 — "Stale-about-stale": the local docs' own warnings outlive their truth** (faq.md "stale 3 ways" is
no longer true; MANAGER L597 SEO note; dead voice.md paths). You cannot trust a "stale" warning without
re-verifying — the exact cost of the append-don't-reconcile habit, and the core argument for Phase 3.

**Direct tie-back to the boat page:** P3's weak-Figma-rule + the unmerged "build from node / verify class
names / stepped-not-flat" guidance means **the installed skill was simultaneously missing the rules that
would have prevented the boat issues AND carrying one that caused them.** Adinda's instinct was right.
